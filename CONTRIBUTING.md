# Contributing to the Docs

Thank you for your interest in improving this documentation!  
Your feedback, suggestions, and contributions are welcome.

## üöÄ How to Contribute

1. **Fork the repository**  
   Click "Fork" on GitHub to create your own copy.

2. **Clone your fork locally**
   ```
   git clone https://github.com/Coockiepickle/Knowledge-Base.git
   ```

3. **Create a new branch**  
   ```
   git checkout -b contrib/your-changes
   ```

4. **Make your changes**  
   - Edit or add Markdown files in the `docs/` directory.
   - Follow the style and structure of existing documentation.
   - Keep explanations clear and examples relevant.
   - Reference or update links where necessary.

5. **Commit and push your changes**
   ```
   git add .
   git commit -m "Describe your changes"
   git push origin contrib/your-changes
   ```

6. **Open a Pull Request**
   - Go to your fork on GitHub and click "Compare & pull request".
   - Describe your changes for the update.
   - Submit your pull request for review.
  
7. **What Next?**
   
  I'll:
  - Review your changes,
  - Test them locally,
> [!NOTE]
> I mean that, for example, if you add a doc to install RustDesk, I'll install RustDesk following your doc
  - Modify and add what I think is missing (if I run into errors or something else),
  - Contact you to discuss the changes I made to know if you accept them,
  - Publish them on my docs with you as the main author.

## üìã Guidelines

- Write clearly and concisely
- Use proper Markdown and [shadcn-theme](https://asiffer.github.io/mkdocs-shadcn/) formatting
- Check spelling and grammar
- Do not include sensitive or proprietary information
- Reference issues or discussions when relevant
- Include a contact info to discuss (professional/temporary/personal email, social media account...)
